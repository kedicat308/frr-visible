阶段01

trace-aggregator 拉 Loki 的 frr-txn 锚点,按提交时刻 ≤ 收敛起点 ≤ 提交+窗关联(容 mac↔VM 时钟偏移),给导出的 span 打 txn.id / txn.action / txn.agent。未归因收敛短暂 hold 等锚点到再导出,免重复。

阶段02

遥测同源于 SoT接缝裂缝✓ 修复

发现真实裂缝:fpm/bmp/log 曾带外 vtysh 配置,每次治理 apply 的 frr-reload 把它收敛回 frr.conf 时擦掉了读平面的采集配置。修法:纳入 frr.conf.j2 渲染(逐字匹配 FRR 规范形态免 drift)——frr-reload 从「擦掉它」变为「保护它」。

before apply 后 pe1/ce1 的 bmp/fpm 配置被擦光 · shim 收不到新收敛
after 读/写同源于 git · 治理 apply 后 fpm/bmp/log 存活 · frr-reload 零漂移
阶段03

读平面自愈:reconnect → reconcile多平面互证✓ 实测

syslog 结构上自盲——发现不了自己丢消息,FRR 重启时更是毫无信号。用有对账能力的通道触发没有的:FPM/BMP 拨入重连 = FRR 重启的带内铁证 → 触发 OSPF reconcile;外加默认开 60s 兜底。

shim.log [bmp] bgpd connected → [ospf] reconcile kicked by bmp-reconnect
[fpm] zebra connected → [ospf] reconcile kicked by fpm-reconnect
阶段04

shim 解析 l2vpn evpnRT-2/3/5✓ 实测

BMP 原只解 VPN-IPv4。补 afi=25/safi=70 分支,解码 EVPN NLRI:type-2 MAC/IP、type-3 IMET、type-5 IP-prefix → 相关器事件 + gNMI cache。EVPN 控制面收敛现与 L3VPN 的 vpn-announce 对等可追。

dc evpn t2 aa:c1:ab:90:7b:84/10.20.20.11 · t3 10.255.2.12 · t5 10.20.{10,20,30}.0/24
MAC 通告 churn → trace → Tempo span txn.id=17ef49ca(RD/nh/子网全对)
阶段05

前缀因果链 + 内核作证lineage✓ 实测

netlink 补订 RTNLGRP_*_ROUTE——内核对「实际装了什么」的独立作证,区别于 FPM「zebra 打算装什么」。lineage 包按前缀把 BMP-in → FPM → kernel 三平面 join,reconcile 检 install-gap(推了内核没认)。

/lineage carrier 10.255.3.102/32 · dc 10.20.20.0/24 = rib-in + fpm + kernel 三平面全链
健康网 gap=0 · install-gap 已 armed(FPM present · kernel 缺 · 超 grace → 根因 span)

核心原则 · 四平面互证

每个平面替别的平面作证,单平面永远做不到

观测的可信度不来自某一路数据全,而来自平面之间能互相证伪:一个结构上自盲的通道,靠另一个有对账能力的通道来暴露它的盲区。

rib-in 有 · fpm 无 → 策略丢

邻居发来了,但没进 Loc-RIB / 没被选中 —— 死在入向策略或选路。

fpm 有 · kernel 无 → 装失败

zebra 说装了,内核 grace 内没确认 —— nexthop 不可解析 / 表满 / MPLS 未开。install-gap。

诚实标注

install-gap 已 armed,未强制触发 —— 检测逻辑与三平面 join 已实测(健康网 gap=0),但不在生产 FIB 上人为制造安装失败来演示。 EVPN 因果链的 rib-in 来自 type-5 IP-prefix(它才是 IP 路由);type-2 MAC/type-3 IMET 是 L2 层,进 trace 但不进 IP 前缀 lineage。 下一步可选 —— BMP pre/post-policy 拆分(rib-in 与 Loc-RIB 精确分层)、adj-rib-out(转发给下游那跳)、install-gap 的受控演示。
